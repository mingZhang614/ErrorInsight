
# ErrorInsight Framework for MLLM Evaluation

## Overview

**ErrorInsight** is a multimodal agent framework designed for error diagnosis in **Multimodal Large Language Models (MLLMs)**. It performs **case-level error diagnosis** by identifying the causes of model failures, such as misperceptions, incorrect reasoning, and other cognitive failures. This framework operates on the **ChallengeBench** dataset, which consists of high-difficulty image-question pairs curated from multiple existing MLLM benchmarks.

The **ErrorInsight** framework analyzes model outputs in a structured way, using a **decision-tree-based probing** process and generating **natural-language explanations** for the underlying causes of errors. This allows for more interpretable, reproducible, and actionable evaluations of MLLM performance beyond aggregate accuracy scores.

## Dataset

The **ChallengeBench** dataset, which serves as the diagnostic substrate for the **ErrorInsight** framework, is available on **HuggingFace**. You can find it [here](https://huggingface.co/datasets/Anonymous02345/ChallengeBench).

This dataset includes 1,576 manually verified image-question pairs categorized into 10 capability dimensions and 36 subcategories. It covers various tasks such as **Basic Visual Perception**, **Reasoning**, **Object Recognition**, **Illusion and Hallucination**, and more.

## How to Use

### 1. Dataset and Image Files

Make sure to download the **ChallengeBench dataset** and place the **image files** in the appropriate directory.

- **Dataset JSON**: You should download the **ChallengeBench dataset** from HuggingFace (refer to the provided dataset link above).
- **Images**: Place the images referenced in the dataset JSON file in the `./images` folder.

### 2. Running the Framework

Once the dataset and images are prepared, run the `ErrorInsightFramework` on your selected **Gemini-2.5-Pro** model (or any other model). Below is an example of how to run the evaluation process:

```python
from base_agent import BaseAgent
from system_prompts import sys_prompts_list
from google import genai
from google.genai import types
from io import BytesIO

# Initialize your model and agent
SYSTEM_PROMPT = sys_prompts_list[0]['prompt']
diag_agent = BaseAgent(system_prompt=SYSTEM_PROMPT)
test_model = EvaluatedMLLM(model_name="Gemini-2.5-Pro")

# Set up the framework with paths to dataset and images
framework = ErrorInsightFramework(
    dataset_path="ChallengeBench.json",
    image_dir="./images",
    diagnostic_agent=diag_agent
)

# Run the error diagnosis
framework.run(test_model)
```

### 3. Model Inference

The **`EvaluatedMLLM`** class is used to handle inference for the **Gemini-2.5-Pro** model (or other supported models). It uses the **Google GenAI API** to process image-question pairs and generate answers based on the model's capabilities.

### 4. Error Diagnosis Process

1. **Main Question Evaluation**: The model is first tested on the main question in each image-question pair. If the answer is correct, the process is skipped. If incorrect, probing begins.

2. **Probing**: A series of probe questions are asked based on the **decision-tree** outlined in the `merge_data_path`. Each probe helps identify specific cognitive weaknesses in the model, such as visual recognition or reasoning errors.

3. **Error Diagnosis**: The diagnostic agent (`BaseAgent`) generates natural-language explanations for the error causes, which are then recorded and saved.

4. **Progress Tracking**: The progress of processed files is tracked and saved in the **progress file** to avoid reprocessing.

### 5. Output

The results of the error diagnosis are stored in a JSON file (`error_insight_results.json`), which contains the following information for each image:

- **Image File**: The filename of the image.
- **Ability**: The ability category of the question (e.g., Basic Visual Perception).
- **Diagnosis**: The model's error diagnosis generated by the agent.

### Example Output:

```json
[
    {
        "image_file": "image_984.png",
        "ability": "Basic Visual Perception",
        "diagnosis": "Reasoning Error: The model failed to integrate visual information with logical reasoning in the final answer."
    },
    {
        "image_file": "image_985.png",
        "ability": "Color Perception",
        "diagnosis": "Perception Error: The model incorrectly recognized the color contrast in the image."
    }
]
```

## Files Provided

1. **`base_agent.py`**: Contains the `BaseAgent` class used to generate error diagnoses.
2. **`system_prompts.py`**: Contains predefined system prompts to guide the agent's reasoning during the error diagnosis process.
3. **`ChallengeBench.json`**: The ChallengeBench dataset, containing image-question pairs and probe information.
4. **`ErrorInsight Results`**: The final error diagnosis results will be saved in a JSON file for further analysis.
